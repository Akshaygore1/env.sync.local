Usage guide

Built for developer velocity

From one-liners to mode switching and force-pull recovery, env-sync keeps your team fast while preserving deterministic local secrets behavior.

Quick commands

env-sync           # sync
env-sync status
env-sync discover
env-sync --help

Mode management

env-sync mode get
env-sync mode set trusted-owner-ssh
env-sync mode set secure-peer
env-sync mode set dev-plaintext-http
env-sync mode set secure-peer --yes
env-sync mode set trusted-owner-ssh --prune-old-material --yes

Peer management

secure-peer mode: invite, request access, approve/revoke, trust inspection.

env-sync peer invite --expires 1h
env-sync peer request-access --to hostname.local --token <token>
env-sync peer list --pending
env-sync peer approve new-host.local
env-sync peer revoke compromised-host.local
env-sync peer trust show hostname.local

trusted-owner mode: SSH-reachable peers can sync without explicit approval.

Secret and sync operations

env-sync add KEY="value"
env-sync remove KEY
env-sync list
env-sync show KEY
eval "$(env-sync load 2>/dev/null)"

env-sync sync
env-sync sync hostname.local
env-sync sync --force-pull hostname.local
env-sync sync --dry-run

--force-pull fully overwrites local secrets from the selected host and creates a backup first.

Service, cron, backup, and troubleshooting

env-sync serve -d
env-sync service stop
env-sync service restart
env-sync service uninstall

env-sync cron --install --interval 30
env-sync cron --show
env-sync cron --remove

env-sync restore
env-sync restore 1

tail -f ~/.config/env-sync/logs/env-sync.log
